## 可信账号

### RFM

RFM 模型是衡量客户价值和客户创造利益能力的重要工具，我们可以通过用户最近消费情况 (Recency)、消费频率（Frequency）、消费金额 (Monetary) 三个维度描述出客户一段时间内的价值状况，基于此可以针对用户进行精细化的营销和风控，在重要用户的支付体验保障，优质客户的流失挽回方面提供了有力的数据支持。

|     客户     | 消费情况 Recency | 消费频率 Frequency | 消费金额 Monetary |
| :----------: | :--------------: | :----------------: | :---------------: |
| 重要价值客户 |      **Y**       |       **Y**        |       **Y**       |
| 重要发展客户 |      **Y**       |                    |       **Y**       |
| 重要保持客户 |                  |       **Y**        |       **Y**       |
| 重要挽留客户 |                  |                    |       **Y**       |
| 一般价值客户 |      **Y**       |       **Y**        |                   |
| 一般发展客户 |      **Y**       |                    |                   |
| 一般保持客户 |                  |       **Y**        |                   |
| 一般挽留客户 |                  |                    |                   |

参考RFM模型，从历史交易信息中提取 R、F、M数据等进行分析，产出有效区间窗口下用户的设备、网络、地域关系强弱的数据，总结为验证可信、稳定性可信、交易可信三个范畴，可充分描述客户的正面指标。

### RFMO

RFM具有滞后性，无法及时阻断突发风险，所以可在RFM基础上加上O，即对客户做风险评估。**O（Occasional）**是反映偶然性的负向数据，包括黑名单、模型评分（账户质量、设备质量）、高危操作等，可有效降低交易风险。

负向特征：风险模型、高危操作、风险名单
正向特征：交易可信、稳定性可信、验证可信

结合正负特征，对可信账号动态划分成六个级别：潜在风险账号、灰度可信账号、弱可信账号、普通可信账号、次强可信账号、强可信账号。其中前三级作为中高可信层，在免密引导、绑卡、支付、忘记密码曝光等环节降低打扰，后三级基于风险较高和账户质量较低优先触发验证、保障账户安全。
